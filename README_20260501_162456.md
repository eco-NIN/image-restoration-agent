# 改动日志与实施见解（20260501_162456）

## 约束声明
- 本次仅改动 image-restoration-agent 目录。
- 4KAgent 目录未做任何代码修改。

## Step 1：工作台按钮布局优化
- 修改文件：frontend/src/pages/WorkbenchPage.jsx
- 改动内容：
  - 未运行时“开始处理”按钮改为单行居中/满宽显示（移除栅格分列）。
  - 运行时“结束当前”同样保持满宽展示。
- 见解：单按钮时不应该占左侧半列，满宽更符合视觉重心。

## Step 2：重启后取消状态纠正
- 修改文件：backend/app/main.py
- 改动内容：
  - 新增启动时状态校正：
    - cancelling -> cancelled
    - running + cancel_requested=1 -> cancelled
    - running + cancel_requested=0 -> failed（服务重启中断）
- 见解：服务重启会丢失运行中的子进程，需要在启动时对数据库中遗留状态收敛。

## Step 3：工作台状态同步增强
- 修改文件：frontend/src/pages/WorkbenchPage.jsx
- 改动内容：
  - 页面恢复缓存后，增加一次批量拉取服务端状态，同步任务真实状态。
- 见解：缓存状态可能过期，启动时同步可避免“取消中”或失败状态长时间保留。
