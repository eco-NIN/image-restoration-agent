## 关于你提出的 2/3/4 点见解（实施建议）

### 2）network_turbo 是否要写进代码
结论：不建议写进代码。应在 shell/服务启动层执行一次环境注入。
- 推荐做法：
  1. curl -sSL -o /etc/network_turbo ...
  2. chmod +x /etc/network_turbo
  3. source /etc/network_turbo
  4. 再启动 uvicorn / 相关服务
- 原因：该能力属于主机网络环境优化，不是应用业务逻辑；放到代码中会耦合部署环境且存在权限问题。

### 3）uvicorn 终端展示推理命令
已落地：backend/app/main.py 中已加入 logger.info 输出推理命令和 cwd。你在 uvicorn 控制台可直接看到每个 task 的调用命令。

### 4）Set5 指标评估（4KAgent + HMA/HAT-L/DRCT/X-Restormer/SwinIR）
建议拆两步：
- 第一步：分别生成各方法在 Set5 的输出图（统一放到可评估目录，命名与 GT 对齐）。
- 第二步：使用 4KAgent/eval 下脚本做指标统计。

当前仓库中可直接利用：
- Classic SR 指标脚本：4KAgent/eval/test_metrics_classic.py（README 已注明 crop_border=4）
- 非参考指标脚本：4KAgent/eval/test_metrics_nr.py 或 test_metrics_nr_low_gpu.py

你需要准备：
- Set5 GT 目录
- 各方法输出目录（ExpSR-s4-F、HMA、HAT-L、DRCT、X-Restormer、SwinIR）
- 确保输出文件名与 GT 一一对应

然后按脚本参数约定分别跑：
- 参考类：PSNR / SSIM / LPIPS / DISTS / FID
- 非参考类：NIQE / CLIPIQA / MUSIQ / MANIQA
