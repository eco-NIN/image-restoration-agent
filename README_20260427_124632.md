# Set5 五张图评估实操说明（20260427_124632）

目标：你现在有 5 张图，想评估 4KAgent 的结果分数。
本说明给你最小可执行流程：怎么放图、怎么跑命令、怎么看结果。

注意：本说明只操作 image-restoration-agent 下的评估目录，不改 4KAgent 代码。

## 1. 先确定你的两类图片
你至少需要两套图：
1. GT 真值图（高质量参考图）
2. 4KAgent 输出图（待评估图）

如果你只有“低清输入图”，那还不能算参考指标（PSNR/SSIM/LPIPS/DISTS/FID），因为这些需要 GT。

## 2. 推荐目录结构（5 张图）
在 image-restoration-agent 下新建如下目录：

/gz-data/projects2/image-restoration-agent/eval_set5_5/
  gt/
  sr_4kagent_expsr_s4_f/
  logs/

要求：
1. gt 和 sr_4kagent_expsr_s4_f 中的文件名必须一一对应
2. 全部使用 png 后缀
3. 两边文件数量一致（都为 5）

示例文件名：
img001.png
img002.png
img003.png
img004.png
img005.png

## 3. 放图后先做一致性检查
在终端执行：

cd /gz-data/projects2/image-restoration-agent/eval_set5_5
ls gt | sort > /tmp/gt_list.txt
ls sr_4kagent_expsr_s4_f | sort > /tmp/sr_list.txt
diff /tmp/gt_list.txt /tmp/sr_list.txt

如果 diff 没输出，说明文件名已对齐。

## 4. 跑参考类指标（PSNR/SSIM/LPIPS/DISTS/FID）
使用脚本：[4KAgent/eval/test_metrics_classic.py](../4KAgent/eval/test_metrics_classic.py)

命令：

cd /gz-data/projects2/4KAgent
python eval/test_metrics_classic.py \
  --inp_imgs /gz-data/projects2/image-restoration-agent/eval_set5_5/sr_4kagent_expsr_s4_f \
  --gt_imgs /gz-data/projects2/image-restoration-agent/eval_set5_5/gt \
  --log /gz-data/projects2/image-restoration-agent/eval_set5_5/logs \
  --crop_border 4

cd /gz-data/projects2/4KAgent
python eval/test_metrics_classic.py \
  --inp_imgs /gz-data/projects2/image-restoration-agent/eval_set5_5/sr_4kagent_expsr_s4_p \
  --gt_imgs /gz-data/projects2/image-restoration-agent/eval_set5_5/gt \
  --log /gz-data/projects2/image-restoration-agent/eval_set5_5/logs \
  --crop_border 4


说明：
1. Classic SR 场景建议 crop_border=4
2. 日志会输出每张图分数和平均分
3. FID 会单独计算一次目录级别分数

## 5. 跑非参考指标（NIQE/CLIPIQA/MUSIQ/MANIQA）
方案 A（显存够）：[4KAgent/eval/test_metrics_nr.py](../4KAgent/eval/test_metrics_nr.py)

cd /gz-data/projects2/4KAgent
python eval/test_metrics_nr.py \
  --inp_imgs /gz-data/projects2/image-restoration-agent/eval_set5_5/sr_4kagent_expsr_s4_f \
  --log /gz-data/projects2/image-restoration-agent/eval_set5_5/logs \
  --crop_border 0

方案 B（显存紧张）：[4KAgent/eval/test_metrics_nr_low_gpu.py](../4KAgent/eval/test_metrics_nr_low_gpu.py)

cd /gz-data/projects2/4KAgent
python eval/test_metrics_nr_low_gpu.py \
  --inp_imgs /gz-data/projects2/image-restoration-agent/eval_set5_5/sr_4kagent_expsr_s4_f \
  --log /gz-data/projects2/image-restoration-agent/eval_set5_5/logs \

建议：先跑标准 NR 版本（test_metrics_nr.py）；只有显存不足或 OOM 时，再切换 low_gpu 版本。

## 5.1 论文对齐检查清单（Set5）
如果你要尽量对齐论文表格中的数值，请逐条确认：
1. 数据一致：与论文相同的 Set5 图像与同一缩放倍率（常见 x4）。
2. 文件对齐：GT 与结果图同名一一对应，且都为 png。
3. 评测参数一致：参考指标常用 crop_border=4，NR 常用 crop_border=0。
4. 结果来源一致：4KAgent 输出来自同一 checkpoint、同一推理配置。
5. 颜色空间一致：确认是否按 RGB 或 Y 通道统计（与论文设定保持一致）。
6. FID 认知：5 张图样本很小，FID 波动会明显大于大数据集。

## 8. 常见报错与处理
1. 报错：数量不一致
   处理：检查 gt 与 sr 文件数量、文件名。

2. 报错：读取失败
   处理：确保是 png，且文件未损坏。

3. 报错：CUDA OOM
   处理：改用 test_metrics_nr_low_gpu.py，或减少并发任务。

4. 指标异常（例如 PSNR 很低）
   处理：优先检查是否文件名错配，是否把输入低清图误当成 GT。

## 9. 一句话最短流程
1. 把 5 张 GT 放到 gt，把 5 张 4KAgent 输出放到 sr_4kagent_expsr_s4_f，文件名同名同序。
2. 跑 test_metrics_classic.py 得到参考类指标。
3. 先跑 test_metrics_nr.py 得到非参考类指标；若 OOM 再改跑 test_metrics_nr_low_gpu.py。
4. 到 logs 里看平均分。
