# 改动日志与实施见解（20260427_225020）

## 约束声明
- 本次仅改动 image-restoration-agent 目录。
- 4KAgent 目录未做任何代码修改。

## Step 1：后端任务 ID 改为时间命名，上传文件按“原名+时间”命名
- 修改文件：backend/app/main.py
- 主要改动：
  - 新增 now_token / make_task_id / make_batch_id。
  - 任务 ID 改为 `task_YYYYMMDD_HHMMSS_mmm_XXXX`。
  - 批次 ID 改为 `batch_YYYYMMDD_HHMMSS_mmm_XXXX`。
  - 上传文件保存名改为 `原文件名stem + 时间戳 + 后缀`。
- 设计见解：
  - 时间命名便于排查问题、人工检索与归档。
  - 追加短随机后缀避免高并发同毫秒冲突。

## Step 2：文件夹上传在后端保留目录结构
- 修改文件：backend/app/main.py
- 主要改动：
  - 批量接口新增 form 字段 `image_paths`（前端逐文件传 `webkitRelativePath`）。
  - 后端按 `UPLOAD_DIR/batch_id/相对路径父目录/重命名文件` 保存。
  - 数据库存储 `relative_path / upload_group / source_type`，用于历史分组和展示。
- 设计见解：
  - 文件夹结构保留后，后续打包转存、回溯来源更容易。
  - 同批次统一放到 upload_group 下，避免平铺造成混乱。

## Step 3：后端支持主动取消任务
- 修改文件：backend/app/main.py
- 主要改动：
  - 新增 `POST /api/tasks/{task_id}/cancel`。
  - 新增 `cancel_requested` 字段与 ACTIVE_PROCESSES 进程表。
  - `run_4kagent_task` 在运行时检测取消请求，terminate 子进程并落状态 `cancelled`。
- 设计见解：
  - 取消是“异步终止”流程，先置 `cancelling`，最终收敛到 `cancelled`。
  - 同时记录系统日志，便于解释任务为何提前结束。

## Step 4：任务状态与历史接口扩展（支持单图/批次/文件夹差异化）
- 修改文件：backend/app/main.py
- 主要改动：
  - `/api/tasks/{task_id}` 返回新增字段：fileName、originalFileName、relativePath、sourceType、uploadGroup。
  - `/api/history` 从扁平数组升级为“单图条目 + 分组条目（folder/batch）”。
  - 分组条目内含 items 子列表，前端可折叠展开。
- 设计见解：
  - 历史接口直接返回分组结构，前端渲染逻辑更稳定，不必再做复杂二次归并。

## Step 5：前端 API 层补齐批量路径和取消能力
- 修改文件：frontend/src/services/apiClient.js
- 主要改动：
  - `submitRestoreBatchTask` 追加 `image_paths`。
  - 新增 `cancelTask(taskId)`。
- 设计见解：
  - API 层统一封装后，页面逻辑可复用且错误处理一致。

## Step 6：工作台页面重构（满足 1/2/4/5/6/7）
